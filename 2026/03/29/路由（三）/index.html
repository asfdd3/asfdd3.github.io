notifier 触发路由变化

发表于 2026-03-29 分类于 网络协议栈源码学习 阅读次数： Valine：

引言

当网络设备状态发生变化（例如设备up down ，link up link down）或者一个网络设备的IP地址发生变化，路由子系统会通过notifier收到事件，上述两种事件分别会调用到fib_netdev_event 和 fib_inetaddr_event 来处理相应的事件。

fib_netdev_event() 是路由子系统监听网卡事件后的处理函数。当设备被注销、网卡 up/down、链路断开或恢复，内核就会通过 notifier 通知路由子系统。拿到事件以后，会去判断这个事件会不会影响当前设备相关的路由如果是up事件，就把这个网卡上 IP 地址对应的路由重新补进去；如果网卡 down 了或者设备被删除了，就把依赖这个设备的下一跳和路由标记成不可用；如果只是链路状态变了，就根据是 link up 还是 link down 去恢复或者禁用相关 next hop。

fib_add_ifaddr() 本质上是 给一个新配置的 IP 地址补齐它该有的所有路由。先加一条 local 路由（保证发往这个 IP 的包会被当成本地接收），如果网卡已经 up，则根据情况继续添加路由，例如如果是一个正常网段地址（例如24位掩码），还会自动加一条 直连路由，同时还会补一条网络广播地址的路由。总体来说，这个函数就是把一个 IP 地址从配置变成在路由层真正可用。

fib_inetaddr_event() 是路由子系统注册的 IP 地址通知回调函数，用来在 IP 地址新增或删除时，同步对 FIB 路由进行添加或清理。具体实现如下所示：

fib_inetaddr_event() 核心思想就是 监听 IP 地址变化，然后同步更新路由表 。当一个 IP 被添加时，它会调用 fib_add_ifaddr() 自动补齐这个地址对应的各种路由，当 IP 被删除时，则通过 fib_del_ifaddr() 把相关路由清掉。如果这个设备上的最后一个 IP 也被删掉了，还还会把这个网卡在路由层彻底“下线”。上述fib_add_ifaddr在上文已经分析过，这里重点关注fib_del_ifaddr删除路由的过程，具体代码如下所示：

fib_del_ifaddr() 核心思想是 删除一个 IPv4 地址时，负责把它牵连到的那一整套路由删掉。和新增地址不同，删除地址不能直接粗暴删，因为内核还得先判断这个地址对应的 local 路由、广播路由、子网前缀路由，是不是还被同接口上的其他地址共用，尤其还要考虑 primary / secondary 地址之间的关系。所以这个函数一开始先处理 secondary 地址归属到哪个 primary，再把该删的前缀路由先删掉；接着遍历同一网卡上的其他地址，检查当前要删的 local、broadcast、网络广播等路由是否仍然有人在用，把结果记在 ok 标志里，最后只删除那些已经没人共享的广播路由和 local 路由。
