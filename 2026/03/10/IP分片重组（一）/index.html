IP分片与重组模块初始化

发表于 2026-03-10 更新于 2026-03-11 分类于 网络协议栈源码学习 阅读次数： Valine：

引言

IP 分片重组可以理解为：当一个 IP 数据包太大，超过链路的 MTU 时，发送端会把它拆成多个小的分片（fragment） 发出去；这些分片在网络中是独立传输、可乱序到达的。接收端的 IP 协议栈需要根据 源 IP、目的 IP、协议号和 IP ID 等信息，把属于同一个原始数据包的分片暂时缓存起来，并按照偏移量重新拼接成完整的报文，这个过程就叫 IP 分片重组（IP reassembly）。

ipfrag_init() 是 Linux 内核 IPv4 分片重组模块的初始化函数（在inet_init中被调用）。主要作用是 **设置 IP 分片重组所使用的必要参数和结构 ，并注册每个网络命名空间的初始化函数。完成这些初始化之后，当内核在收包路径中遇到 IP 分片时，就可以通过这些已经建立好的结构去 创建分片队列、缓存 frag，并在收齐后完成重组。

这段 ip_expire() 是 IPv4 分片重组队列的超时处理函数，当某个 IP 报文的分片在规定时间内没有收齐时，就会调用这个函数。函数首先找到对应的分片队列 ipq ，如果队列已经完成重组则直接退出，否则将该队列标记为丢弃并从分片哈希表中删除，同时更新分片重组失败和超时的统计计数。如果该报文已经收到了 第一个分片，函数会取出队列中的第一个 skb，重新查路由并检查该报文是否原本是发往本机的，如果是，则发送一个 ICMP 报文通知发送端。最后释放相关的 skb 和分片队列资源。总的来说，这个函数的作用就是当 IP 分片长时间收不齐时，清理分片队列并在需要时向发送端返回 ICMP 超时通知。

rhashtable_init() 的作用是完成rhashtable 的初始化。函数首先检查参数是否合法（必须提供固定长度 key 或自定义对象 hash，同时若提供对象 hash 还必须提供比较函数），然后清空 rhashtable 结构并初始化锁，接着保存用户传入的配置参数。随后对哈希表进行规范处理，再计算初始桶数量 size。之后调用 bucket_table_alloc() 分配底层的 bucket table（桶数组） 作为哈希表的实际存储结构，如果分配失败则退化为最小规模并强制重新分配。最后将元素计数初始化为 0。
