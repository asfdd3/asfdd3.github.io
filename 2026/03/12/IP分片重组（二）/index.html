IP分片

发表于 2026-03-12 更新于 2026-03-15 分类于 网络协议栈源码学习 阅读次数： Valine：

在网络中，一个报文从源主机发送到目的主机的过程中，可能会经过多条不同类型的网络链路，而这些链路所能承载的最大传输单元MTU并不一定相同。当一个 IP报文的大小超过某条链路允许的 MTU 时，如果直接发送将无法通过该链路。为了解决这一问题，IPv4 协议设计了 IP 分片机制：在必要时将一个较大的 IP 数据报拆分成多个较小的分片分别传输，每个分片都携带自己的 IP 头部信息，并在最终目的主机处重新组装成原始数据报。在这一过程中，IP 头中的 Identification、Fragment Offset 以及 MF（More Fragments）标志位 用于标识分片之间的关系和顺序，从而保证数据能够正确地重组。

快速路径

快速路径 的核心思想是如果原始 skb 已经以 frag_list 的形式组织了多个子 skb（通常来自上层构造的分段结构），并且这些子 skb 的大小、8 字节对齐、头部空间以及引用状态都满足分片要求，那么内核就不再重新复制数据进行分片，而是直接复用这些已有的子 skb 作为 IP 分片。具体流程是：首先检查第一个 skb 和所有 frag_list 中的 skb 是否满足 长度 ≤ mtu、非克隆、8 字节对齐、头部空间足够 等条件，如果全部合法，就通过 ip_fraglist_init() 初始化遍历状态，然后在循环中依次为每个子 skb 调整 IP 分片字段（frag_off、MF、offset 等），随后直接调用 output() 将这些 skb 逐个发送。整个过程中不需要重新分配 skb 或复制 payload，只是修改头部并发送，从而避免数据拷贝，提高分片性能。

慢速路径

在慢速路径中，内核不再复用 frag_list，是通过复制数据的方式重新构造每一个 IP 分片。首先调用 ip_frag_init() 初始化分片状态，然后在循环中不断调用 ip_frag_next() 从原始 skb 中按 MTU 切出一段 payload，创建新的分片 skb，并更新分片偏移、MF 标志等字段，随后调用 output() 将该分片发送出去。循环持续直到所有数据都被切分并发送完成，最后释放原始的大 skb。
