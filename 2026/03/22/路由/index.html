路由子系统的初始化

发表于 2026-03-22 分类于 网络协议栈源码学习 阅读次数： Valine：

引言

Linux 网络协议栈中，路由表是路由子系统的核心。与传统认知中查路由表的简单模型不同，Linux 内核中的路由实现由多个子系统协同完成，包括 FIB路由表、策略路由以及基于前缀树结构。本文旨在从源码出发，分析路由子系统的初始化流程。

ip_rt_init为初始化路由子系统的函数之一，在网络子系统初始化函数inet_init中被调用，主要逻辑初始化路由子系统，其中也涉及到一些非路由子系统的初始化，具体代码如下所示：

ip_rt_init 首先构建 IP ID hash表，用于 IP id 的生成（主要给 UDP使用）；然后初始化 每 CPU 的 rt_uncached_list，接着创建 rtable 的 slab 缓存），并初始化 dst_entry 计数器，之后进入功能模块初始化：调用 devinet_init() 初始化 IPv4 地址管理，ip_fib_init() 初始化 FIB 路由表和策略路由框架；再创建 /proc/net/stat/rt_cache 等接口用于统计，最后通过多个 register_pernet_subsys 把 路由 sysctl、默认路由参数等按 net namespace 维度注册，确保每个 netns 都有独立的路由环境。

devinet_init 主要是 初始化 IPv4 地址管理子系统，把 IPv4 挂到 netdevice 通知链能够正确响应网卡状态变化，通过 rtnetlink 注册 NEW/DEL/GETADDR 等接口，使用户态（如 ip addr）能够增删查 IPv4 地址和配置网络参数。
