IP选项处理（一）

发表于 2026-03-05 更新于 2026-03-04 分类于 网络协议栈源码学习 阅读次数： Valine：

引言

在 IPv4 协议设计之初，协议头部被设计为“固定部分 + 可选扩展”的结构。固定首部长度为 20 字节，而当 IHL（Internet Header Length）字段大于 5 时，表示在固定首部之后还存在可选字段（IP Options）。这些选项最多可以扩展 40 字节，使 IPv4 首部最大达到 60 字节。

IP Options 的设计初衷是增强网络层的可观测性与可控性，例如记录路径、指定路由、测量时延等。但随着互联网规模扩大以及安全问题凸显，这些机制逐渐被认为会带来性能下降与安全风险。现代网络中，绝大多数 IPv4 报文都不携带任何选项，而带有 Options 的报文通常会触发路由器或操作系统的慢路径处理。

选项结束（End of Option List）

核心作用： 标记 IP 选项列表结束。
使用现状： 仍可能出现（仅用于填充）。
说明： 占 1 字节，用于提前终止选项解析或做对齐填充，不承载实际网络语义。

空操作（No Operation）

核心作用： 用于选项字段对齐。
使用现状： 仍可能出现（用于对齐）。
说明： 占 1 字节，通常用于将后续选项对齐到 4 字节边界，本身不具备功能性含义。

记录路由（Record Route）

核心作用： 记录报文经过的路由器接口地址。
使用现状： 基本禁用。
说明： 报文在转发过程中，每经过一个路由器就写入其接口 IP。受长度限制，最多记录约 9 个地址。可能泄露网络拓扑信息，因此在现代网络中已基本关闭。

松散源路由（Loose Source Route）

核心作用： 指定报文必须经过的部分路径节点。
使用现状： 基本禁用。
说明： 发送方可以指定若干中间路由节点，但允许中间路径自由选择。属于路径控制机制，存在绕过安全策略的风险。

严格源路由（Strict Source Route）

核心作用： 严格指定完整转发路径。
使用现状： 已废弃。
说明： 报文必须严格按照发送方指定的节点顺序转发，否则丢弃。安全风险极高，现代系统通常直接禁用。

时间戳（Timestamp）

核心作用： 记录转发过程中的时间信息。
使用现状： 基本不用。
说明： 支持仅记录时间、记录 IP+时间、仅指定地址记录时间三种模式。最初用于路径时延分析，但实际部署中极少使用。

安全选项（Security）

核心作用： 标记报文安全级别。
使用现状： 已废弃。
说明： 主要用于早期特定安全或军用网络环境，在公共互联网中几乎不存在。

流标识（Stream ID）

核心作用： 区分不同数据流。
使用现状： 已废弃。
说明： 早期实验性选项，未形成广泛应用。

路由器警告（Router Alert）

核心作用： 通知路由器必须检查该报文。
使用现状： 仍可能出现。
说明： 即使路由器不是最终目的地址，也必须对该报文进行额外处理。常用于 IGMP、RSVP 等控制协议，是现代网络中仍可能看到的少数 IP 选项之一。

IP选项的验证与定位

以上述处理源路由选项为例，在遍历 IP Options 时，首先对选项的基本合法性进行检查，包括：

  • 选项长度 optlen 必须合法（至少包含 type + length + pointer）。
  • pointer 字段必须 ≥ 4，因为前 3 字节已经被 type/len/pointer 占用。
  • 一个 IP 头中 只能存在一个源路由选项，否则判为非法。

在 发送路径（skb == NULL） 下，还会进一步校验：

  • pointer 必须等于 4（表示从第一个地址开始）。
  • optlen 至少为 7（包含一个 IPv4 地址）。
  • 地址列表必须 4 字节对齐。

如果校验通过：

  1. 将第一个路由地址复制到 opt->faddr，作为下一跳地址。
  2. 如果存在多个地址，会通过 memmove() 将剩余地址前移，便于后续处理。
  3. 设置 opt->is_strictroute 标识当前是否为 严格源路由（SSRR）。
  4. 记录源路由选项在 IP 头中的偏移 opt->srr，后续在 ip_options_build() 构造报文时会使用。

整体来说，这段代码完成了 源路由 IP Option 的合法性检查、下一跳地址提取以及内核状态标记。为后续构造数据包等场景做好准备工作。
