inet_frag_queue_insert() 的作用就是**把新的 IP 分片按 offset 插入到当前的重组队列里，并检查是否有重复或重叠。**如果这是 第一个分片，就创建一个新的分片 run；如果新分片 在当前最后一个分片之后，通常直接追加到尾部（连续就拼在一起，不连续就新建一个 run）；如果分片 落在中间位置，就通过 红黑树二分查找找到正确的位置插入。

ip_frag_reasm 负责 把已经收齐的所有 IP 分片拼接成一个完整的数据报。首先 将当前分片队列从哈希表中移除并停止重组定时器，接着调用 inet_frag_reasm_prepare() 选择当前收到的分片 skb 作为最终报文的头 skb，再通过 inet_frag_reasm_finish() 按offset 将所有分片的数据拼接到该 skb 中完成重组。最后更新 IP 头字段，设置新的 tot_len，最后调用重新计算 IP 头校验和。

inet_frag_reasm_prepare 主要作用是 确保当前的 skb 成为队列中第一片，并整理分片链表结构，为后续数据拼接做好准备。函数首先检查当前传入的 skb 是否就是偏移最小的分片，如果不是，则通过 skb_clone() 克隆当前 skb ，再将当前 skb 与真正的第一片在红黑树中的位置进行交换。如果第一片本身还带有 frag_list（例如 GRO 聚合产生的 skb），函数会将该 frag_list 拆分到一个新的 clone skb 中，并返回nextp，从而为 inet_frag_reasm_finish() 在后续步骤中按顺序拼接各个分片数据做好结构准备。

inet_frag_reasm_finish 是 IP 重组的核心函数，它负责把重组队列里已经收齐的各个分片 按顺序真正到最终的 head skb 上。函数先从红黑树和 next_frag 链中依次遍历所有后续分片：如果开启了合并优化且 skb_try_coalesce() 成功，就直接把分片的数据页合并进 head；否则就把该分片作为一个独立的 skb 挂到 head 的 frag_list 后面，同时累计更新 head 的 len、data_len 和 truesize。在遍历过程中，它还会处理校验和状态。最后，函数清理红黑树中的节点，修正分片内存记账，把重组完成时间戳等信息写回 head，使这个 head skb 最终成为一个可继续向上传递的重组报文。
