IP重组（一）

发表于 2026-03-15 分类于 网络协议栈源码学习 阅读次数： Valine：

IP 重组是TCP/IP协议栈必须支持的功能。当一个 IP 数据包在传输过程中因为链路 MTU 限制被拆分成多个分片时，这些分片会分别独立在网络中转发，并可能乱序到达目的主机。接收端通过 Identification、Fragment Offset 和 MF 标志 来识别属于同一数据包的分片，并维护一个分片队列，按照offset将各个分片重新拼接成完整的数据报；只有当所有分片都到齐时，报文会被重组并继续交给上层协议处理。

ip_frag_queue() 的核心作用是**把新收到的 IP 分片放入对应的重组队列，并判断是否已经可以完成重组。**当一个新的 IP 分片到达时，内核先检查这个数据报是否已经完成重组，如果已经完成那说明这是重复分片，直接丢弃；如果还没完成，则调用ip_frag_too_far是否超过用户配置的安全距离，接着根据是否是最后一片更新重组队列的总长度，并检查分片是否合法。然后对 skb 做必要的处理（去掉 IP 头、裁剪到正确长度），再调用 inet_frag_queue_insert() 按 offset 把该分片插入到分片队列中。插入成功后更新统计信息，如果 首片和尾片都已经到达，并且收到的数据总长度等于完整报文长度，就调用 ip_frag_reasm() 把所有分片拼接成一个完整的数据包继续向上层协议递交；如果还没收齐，就暂时保存分片等待后续片段到达。

ip_frag_too_far 的作用是检测当前分片是否和之前的分片距离过远，用于防止恶意或异常的分片攻击，如果系统开启了该安全机制（即 peer 和 max_dist 存在），内核会利用 inet_peer 中的一个全局递增计数 rid 来记录分片到达的顺序。每当收到一个新的分片，就增加这个计数，并计算当前队列中 上一次记录的start和当前 end之间的差值。如果这个差值超过了允许的最大距离 max_dist，并且当前重组队列中已经有分片存在，就认为新的分片和之前的分片间隔过大，说明中间可能丢失了大量分片或者出现异常乱序，于是返回 true，触发后续重置或丢弃该重组队列，从而避免攻击者通过大量远距离分片耗尽系统内存。
