ip_rcv_options首先通过 skb_cow() 确保 skb 可写，然后计算并记录 options 长度，并调用 ip_options_compile() 对所有 IP 选项进行合法性检查和解析（如 RR、TS、LSRR、SSRR 等）。如果存在源路由选项，还会检查当前网卡是否允许 source route；若未开启则记录日志并丢包，否则调用 ip_options_rcv_srr() 进一步处理源路由逻辑。

ip_options_rcv_srr会遍历 SRR 选项中的地址列表，逐个取出下一跳地址并重新进行路由查找验证其可达性，如果下一跳仍然是本机地址，则修改 IP 头目的地址继续遍历（相当于本机内部 loopback forward），直到遇到一个需要转发出去的地址为止，此时记录该地址为 nexthop 以供后续转发使用。
