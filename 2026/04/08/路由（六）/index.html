输出方向路由查找

发表于 2026-04-08 分类于 网络协议栈源码学习 阅读次数： Valine：

在 Linux 内核中，当应用层通过 send()等接口发送数据时，内核必须先决定这个包该从哪个接口发、下一跳是谁以及用哪个源地址。因此需要进行输出方向路由查找，通过根据目的地址及相关策略在 FIB 中匹配合适的路由项，将抽象的发往某个 IP转化为具体的发送路径（包括出接口、网关等信息），从而指导后续数据包正确发出。

ip_route_output_key_hash_rcu() 是输出方向路由查找的主核心函数：首先根据 flowi4 里的源地址、目的地址和出接口等信息做前置检查与补全，比如用户显式指定了源地址时会先判断这个源地址是否合法、在某些组播/广播场景下还会直接根据源地址选择发送接口。如果用户指定了输出设备，就先检查这个设备是否存在、是否处于 UP 状态，并在没有源地址时从该设备上挑一个合适的本地地址。正常情况下则调用 fib_lookup (在前文已经已经分析过)去查 FIB 路由表。查到结果后，如果目标其实是本地地址（RTN_LOCAL），就走 loopback 路径，如果是普通单播路由，则通过 fib_select_path() 从查到的路由结果里进一步选出具体的 nexthop、出接口和源地址。最后无论前面走的是特殊分支还是普通查表分支，都会统一落到 __mkroute_output()，把这些查找结果封装成最终可供后续发送流程使用的 rtable。

上述代码的核心逻辑是为输出路由选择一个合适的源 IP 地址，首先优先使用路由项里显式配置的 fib_prefsrc，如果没有就尝试从 nexthop里拿之前缓存的源地址，但前提是这个缓存仍然有效（说明网卡地址没变），如果缓存无效或不存在，就调用 fib_info_update_nhc_saddr() 重新选择一个源地址（本质是通过 inet_select_addr() 从对应网卡上选一个最合适的 IP，比如同网段或满足 scope 的地址），并把结果更新回 nexthop 做缓存。

__mkroute_output() 的作用是把前面路由查找得到的 fib_result 真正封装成一个可发送的 rtable，它先检查输出设备和源/目的地址是否合法，并根据目的地址把路由类型修正成广播、多播或普通单播，接着根据路由类型决定一些标志位以及是否允许缓存，像广播、多播、本地但强制指定接口这几种情况往往会限制缓存行为。然后如果这条路由对应的 fib_info 存在，它会优先尝试复用已有的输出路由缓存，如果缓存不可用，会分配一个 rtable，并根据场景设置默认的输入/输出处理函数，比如多播可能走 ip_mc_output。最后再调用 rt_set_nexthop() 把下一跳、网关、路由缓存信息等继续填进去。

上述rt_set_nexthop具体实现如下所示：

这段 rt_set_nexthop() 的核心作用是把下一跳相关的信息真正写入 rtable，并决定是否进行路由缓存：如果存在 fib_info（典型单播场景），它会先判断是否需要经过网关（有 gateway 就设置 rt_gw），再把路由的 metric 参数填到 dst 里（后面 TCP 发包很多参数从这里取），同接着优先尝试把这条路由缓存到 nexthop，如果缓存成功就直接复用，否则说明这条路不能挂到 nhc 上，就会补一个可用于邻居解析的目标地址，并把该路由加入 uncached_list 做兜底管理。确定下一跳（是否走网关）、填充发送所需参数，并尽量把路由放进缓存以加速后续查找，否则就作为非缓存路由单独管理。
