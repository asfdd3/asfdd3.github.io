IPv4地址相关操作接口

发表于 2026-01-30 更新于 2026-02-04 分类于 网络协议栈源码学习 阅读次数： Valine：

本文将分析内核IP地址是如何被管理及介绍操作IP地址的常用接口。要知道在内核网络协议栈中，IP地址是设置到主机的网络接口，同时一个网口可以有多个IP地址。
net_device可以视作驱动与协议栈沟通的桥梁之一，其中IPv4相关的配置被net_device的in_device结构中，而具体的IP地址，子网掩码，广播地址等信息存放到in_device的in_ifaddr中，具体关系如下所示：

in_device 结构

in_device 是每个网络接口对应的 IPv4 管理结构。当驱动注册net_device时，内核在 inetdev_init() 中为该 net_device 创建一个 in_device，并通过 dev->ip_ptr 与之关联。in_device 本身并不保存具体的 IP 地址，而是维护一个链表，用来管理该网口上的所有 IPv4 地址。同时，in_device 还保存了该接口的 IPv4 行为配置（如 rp_filter、arp_ignore、arp_announce、forwarding 等）。

in_ifaddr结构

in_ifaddr 是 Linux 内核中 一个具体 IPv4 地址的实体对象。当执行 ip addr add ... dev eth0 时，内核并不是把 IP 直接写进网口，而是创建一个 in_ifaddr 结构体，并将其挂到该网口对应的 in_device->ifa_list 链表中。这个结构体中保存了该地址的核心信息，如本机地址、广播地址、子网掩码、作用域、标志位等。具体结构如下所示：

IP 操作函数

inetdev_init

inetdev_init的作用是为一个net_device分配一个in_device结构，当网卡驱动注册设备时，会通过通知链机制调用到该函数（devinet_init中注册）

inetdev_init() 的作用很简单：让一个网口关联管理IP的结构。它为新注册的 net_device 分配一个 in_device，把当前网络命名空间的 IPv4 默认配置拷贝过来，建立与网口的关联，并初始化 ARP 参数、IGMP 多播相关结构以及 /proc/sys/net/ipv4/conf/<dev>/ 下的配置入口。如果开启了转发，还会关闭 LRO （保证转发正确性）。最后把这个 in_device 挂到 dev->ip_ptr 上。

inet_select_addr

inet_select_addr的作用就是在封装数据包的时候如果没有指定数据包的源ip地址，则会调用该接口，根据指定设备目的地址以及地址范围遍历当前netdev并选择一个合适的源ip地址，如果当前设备没有ip地址的话则会fallback获取其他设备的IP地址。

inet_confirm_addr

inet_confirm_addr用于在给定参数的条件下找到一个属于本机的IP地址，如果没有给定具体IP地址还会自动选择一个，**在收到包时，判断是不是需要被处理的（例如arp模块）**会调用该接口。

inet_confirm_addr如果指定了dev则调用confirm_addr_indev判断是否有满足条件的地址，否则遍历所有的网络设备之后调用**confirm_addr_indev 来根据指定网口上，按范围和是否是同一个子网来，返回一个属于本机的 IPv4 地址**

inet_addr_onlink

inet_addr_onlink用于判断 a 是否和这块网卡上的某个 IP 在同一子网

inetdev_by_index

inetdev_by_index作用是通过接口编号，找到这块网卡的 IPv4 地址管理结构。其实就是调用了dev_get_by_index_rcu，路由接口会调用到。

inet_ifa_byprefix

inet_ifa_byprefix作用为在某接口上，按子网前缀 + 掩码查找对应的 IPv4 地址结构并返回IP地址

例如inet_ifa_byprefix(in_dev, 192.168.1.0, 255.255.255.0)

inet_abc_len

inet_abc_len作用为根据指定的IP地址获取默认的掩码长度，配置IP地址会用到
