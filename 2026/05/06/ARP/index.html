ARP初始化（一）

发表于 2026-05-06 更新于 2026-05-08 分类于 网络协议栈源码学习 阅读次数： Valine：

引言

邻居子系统是网络协议栈中负责下一跳邻居管理的通用框架，它主要解决的问题是当内核已经知道数据包该从哪个网卡、发给哪个 IP 后，如何进一步找到对应的二层地址（MAC），以及如何维护这些邻居的可达状态。为此，内核实现了一整套通用机制，包括邻居表管理、状态机、定时器、探测重传、缓存队列以及老化回收等功能。而 ARP 则是这个通用框架在 IPv4 下的具体实现，负责通过 ARP Request/Reply 报文完成 IPv4 地址到 MAC 地址的解析。

ARP 报文格式

字段 含义 常见值

Protocol Type 上层协议类型 0x0800（IPv4）

Target IP 目标 IP 要解析的 IP

初始化

arp_init() 是 Linux 内核 ARP 模块的初始化入口，它主要完成 ARP 与整个网络协议栈的注册和关联。首先通过 neigh_table_init() 将 arp_tbl 注册到通用邻居子系统中。随后通过 dev_add_pack() 注册 ARP 报文接收入口 arp_packet_type，这样网卡收到以太网类型为 0x0806 的 ARP 报文就会进入 arp_rcv() 处理流程。接着 register_pernet_subsys() 会为每个网络命名空间创建对应的 ARP 相关数据结构和 proc/sysctl 接口，使不同 namespace 之间的 ARP 配置和邻居表能够互相隔离。neigh_sysctl_register() 则用于注册邻居子系统相关的 sysctl 参数，方便用户动态调整邻居行为。最后通过 register_netdevice_notifier() 注册网卡事件通知链，这样当网卡 up/down、地址变化或设备注销时，ARP 模块也能同步更新对应的邻居状态和 ARP 表项。

arp_constructor() 是 ARP 邻居表项的初始化函数，在内核新建 struct neighbour 时被调用，它主要负责根据目标 IP 类型和网卡类型初始化这个 neighbour 的行为。函数首先会判断目标地址是单播、多播还是广播，并获取当前网卡对应的 ARP 参数，例如 reachable_time、retrans_time 等。对于多播、广播、回环或 NOARP 设备，由于不需要真正进行 ARP 解析，因此会直接设置 NUD_NOARP 状态并提前生成目标 MAC 地址；而对于普通以太网单播地址，则会根据网卡是否支持二层头缓存来设置 neigh->ops，通常以太网设备会使用 arp_hh_ops，其中包含 neighbour 的 output、connected_output 等发送回调。随后内核会根据当前邻居状态初始化 neigh->output，如果邻居已经有效则直接使用快速发送路径，否则通常会设置为 neigh_resolve_output，这样后续真正发包时如果还不知道 MAC，就会自动触发 ARP 请求进行地址解析。

arp_solicit

arp_solicit() 是 ARP 邻居子系统中的主动探测函数，它的核心作用是构造并发送 ARP Request，用来解析或探测目标邻居的 MAC 地址。具体实现如下所示：

arp_solicit() 是真正负责发送 ARP Request 的核心函数。函数开始会先获取目标 IPv4 地址以及当前邻居的 probe 次数，然后根据 arp_announce 配置决定 ARP Request 使用哪个源 IP，默认情况下会优先使用 skb 中属于本机的源 IP，而更严格的模式则要求源 IP 必须与目标位于同一子网。如果前面没有选出合适的源地址，则通过 inet_select_addr() 从网卡上重新选择一个。接下来函数会根据 neigh->probes 判断当前是单播探测、应用层探测还是广播 ARP 探测，如果之前已经解析过 MAC，则会优先使用旧 MAC 做单播 ARP probe，减少广播；否则会走广播 ARP。最后通过 arp_send_dst() 构造并发送 ARP Request 报文。
