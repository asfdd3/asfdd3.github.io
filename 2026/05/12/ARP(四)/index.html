ARP报文的接收与发送（一）

发表于 2026-05-12 更新于 2026-05-13 分类于 网络协议栈源码学习 阅读次数： Valine：

在 Linux 内核发送路径中，一个 IP 数据包在完成路由查找后，需要经过邻居子系统完成二层地址解析。这个过程的核心入口之一，就是 ip_finish_output2()。该函数位于 IPv4 输出路径的末端，具体代码如下所示：

ip_finish_output2首先调用 ip_neigh_for_gw() 根据路由项中的下一跳信息查找对应的邻居表项 struct neighbour。如果邻居表中已经存在对应项，则直接返回已有 neighbour；如果不存在，则会动态创建一个新的邻居对象，之后向下调用邻居子系统的发包函数入口neigh_output。

ip_neigh_for_gw() 的作用就是确定当前数据包到底应该对哪个 IP 做邻居解析。如果路由结果中标记了需要发往网关，那么真正需要解析 MAC 的其实是网关 IP，因此这里会使用 rt->rt_gw4 查找邻居表；如果没有网关，说明目标就在本地二层网络内，那么直接使用数据包目的 IP daddr 查找邻居即可。ip_neigh_gw4具体实现如下所示：

ip_neigh_gw4() 先根据目标 IP 在 ARP 邻居表中查找对应的 neighbour 表项，如果已经存在就直接返回；如果没找到，则调用 __neigh_create() 动态创建一个新的邻居对象，也就是说邻居表项通常不是提前建立好的，而是在真正发包时发现没有对应 MAC 映射后才动态创建。上述查找邻居表项__ipv4_neigh_lookup_noref实现如下所示：

___neigh_create() 的作用就是创建并插入一个新的邻居表项。它会先通过 neigh_alloc() 申请 struct neighbour，把要解析的 IP 保存到 primary_key，并绑定对应的出接口 dev；然后调用 ARP 自己的 constructor 初始化邻居项，比如设置 neigh_ops、状态和相关参数。接着进入邻居表加锁区域，如果当前表项数量已经超过哈希桶数量，就调用neigh_hash_grow触发哈希表扩容(这里是扩容一倍，并重新hash)；然后根据 IP 和设备计算 hash 值，并再次检查表中是否已经有相同的邻居项，避免并发场景下重复创建。确认没有重复项后，就把新 neighbour 挂到 GC 链表和 hash 桶中。

上述neigh_alloc具体实现如下所示：

上述neigh_alloc() 的作用就是申请并初始化一个新的邻居对象 struct neighbour。在真正分配内存前，内核会先检查当前邻居表项数量是否超过 gc_thresh2/gc_thresh3 阈值，如果邻居表过大就尝试执行 neigh_forced_gc() 强制回收旧表项，避免 ARP 表无限增长。并初始化 ARP 队列、锁、状态机和定时器等基础成员。新邻居项初始状态为 NUD_NONE，发送函数先设置为 neigh_blackhole，表示当前还不能真正发包；同时初始化 arp_queue，用于缓存等待 ARP 解析的数据包。随后还挂一个定时器为后续邻居状态迁移、ARP 重传、可达性提供检测。

neigh_forced_gc() 是邻居表过大时触发的强制回收逻辑。它会遍历邻居表，优先删除那些没有被外部使用的旧表项，比如 NUD_FAILED、NUD_NOARP 或长时间未更新的 neighbour，从而给新的 ARP 表项腾空间，避免邻居表溢出（如果清除不了，也是有可能溢出的！！）。

回到neigh_alloc中，一系列初始化流程完成后，还会为每一个表项挂个定时器，用于处理状态迁移，具体代码如下所示：

neigh_timer_handler() 是邻居项状态机的定时器处理函数，主要负责 ARP 表项的老化、探测和失败处理。它会先读取当前 neighbour 的 nud_state，如果当前是 NUD_REACHABLE，就检查在 reachable_time 内是否收到过确认：如果确认过，就继续保持可达；如果超时但最近还被使用过，就转成 NUD_DELAY，先延迟一段时间再探测；如果太久没使用，就转成 NUD_STALE。如果当前是 NUD_DELAY，说明邻居已经可疑但还没有立即探测，此时如果延迟期间收到了确认，就重新变成 NUD_REACHABLE；否则转成 NUD_PROBE，开始主动发送 ARP 探测。对于 NUD_INCOMPLETE 或 NUD_PROBE 状态，它会周期性调用 neigh_probe()，也就是最终触发 arp_solicit() 发送 ARP 请求；如果探测次数超过限制，就把邻居项标记为 NUD_FAILED，并清理等待队列中的 skb。
