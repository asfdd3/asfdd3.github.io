IGB网卡驱动初始化（五）

发表于 2026-06-23 分类于 IGB网卡驱动 阅读次数： Valine：

igb_open

在前面分析 igb_probe 的时候，已经知道 igb 驱动是如何被 PCI 子系统匹配到、如何分配并初始化 net_device，最后通过 register_netdev() 把网卡注册到内核网络子系统中的。但是probe 完成并不代表网卡已经真正开始收发数据。此时驱动只是把设备“注册好了”，用户态能够看到这个网卡设备，例如通过 ip link 或 ifconfig -a 查看它。只有当用户执行 ip link set ethX up，或者系统网络服务拉起该网卡时，内核才会调用驱动提供的 ndo_open 回调，也就是 igb 驱动中的 igb_open()。它会完成真正运行前的一系列准备工作：申请和初始化描述符资源、配置硬件队列、注册中断、使能NAPI、使能发送接收路径，并最终让网卡具备收发报文的能力。igb_open具体代码如下所示：

__igb_open() 是 首先检查当前网卡是否处于测试状态，避免 ethtool 自检等场景下被错误打开；随后调用 netif_carrier_off() 告诉协议栈当前链路还不可用。接着调用igb_setup_all_tx_resources和igb_setup_all_rx_resources依次分配 TX/RX 描述符资源，拉起 PHY 链路，调用 igb_configure() 完成网卡硬件配置，包括收发路径、RX buffer、RSS、过滤规则等内容，再调用 igb_request_irq() 注册中断处理函数。之后，驱动把实际可用的 TX/RX 队列数量通知给网络协议栈，清除 __IGB_DOWN 状态位，开启 NAPI，清除历史挂起中断，并真正使能网卡中断。最后，驱动启动发送队列，并调度 watchdog 任务去异步检测真实链路状态；只有后续 watchdog 确认 link up 后，才会通过 netif_carrier_on() 通知协议栈链路可用。

igb_setup_all_tx_resources

igb_setup_all_tx_resources() 负责为 igb 网卡的所有发送队列分配软件和硬件所需的 TX 资源。它会遍历 adapter->num_tx_queues 个发送队列，并对每个 tx_ring 调用 igb_setup_tx_resources()。真正的资源分配发生在 igb_setup_tx_resources() 中：首先根据描述符数量 tx_ring->count 分配一组 struct igb_tx_buffer 软件数组，这个数组由驱动使用，用来记录每个 TX 描述符对应的 skb、DMA 地址、长度等软件状态；然后根据 TX 描述符数量计算硬件描述符环的大小。接着通过 dma_alloc_coherent() 分配一块 DMA 一致性内存作为 TX 描述符环，这块内存同时会返回 CPU 可访问的虚拟地址 tx_ring->desc 和网卡硬件可访问的 DMA 地址 tx_ring->dma。因为描述符环是 CPU 和网卡长期共享的数据结构，CPU 要写描述符，网卡要读描述符并回写状态，所以这里使用一致性 DMA 内存。最后驱动把 next_to_use 和 next_to_clean 都初始化为 0，表示发送环从头开始使用。

igb_setup_rx_resources() 负责为单个 RX 队列分配接收方向所需的软件和硬件资源。调用 xdp_rxq_info_reg() 注册当前 RX 队列的元信息，包括所属 net_device、队列编号 queue_index 等，这样后续 XDP 程序在 RX 路径中就能知道自己运行在哪个接收队列上。接着驱动根据 rx_ring->count 分配 struct igb_rx_buffer 软件数组，然后驱动计算 RX 描述符环需要的大小， 分配一块一致性 DMA 内存作为 RX 描述符环。资源分配完成后，驱动把 next_to_alloc、next_to_clean、next_to_use 都初始化为 0，表示 RX 环从头开始使用，其中 next_to_use 表示下一个准备交给硬件的描述符位置，next_to_clean 表示 NAPI poll 接收路径下一个要检查和清理的位置，next_to_alloc 则用于管理 RX buffer 的分配进度。最后，驱动把当前 adapter 上挂载的 XDP 程序记录到 rx_ring->xdp_prog，这样后续该 RX 队列收包时就可以执行对应的 XDP 逻辑。

igb_configure

分配完tx/rx方向的资源并拉起PHY后，会调用igb_configure完成对网卡tx/rx队列，RX buffer、RSS等配置。具体代码如下所示：

igb_configure() 负责把前面已经分配好的软件资源真正配置到网卡硬件中。它首先通过 igb_get_hw_control() 告诉固件当前设备由驱动接管，然后配置接收模式、VLAN、TX 控制、RSS、RX 控制以及 N 元组过滤等硬件功能。接着 igb_configure_tx() 和 igb_configure_rx() 会把 TX/RX 描述符环的 DMA 地址、长度、head/tail 指针以及队列控制参数写入网卡寄存器，使硬件知道每个收发队列的描述符环在哪里。最后，驱动会遍历所有 RX 队列，通过 igb_alloc_rx_buffers() 预先分配接收 buffer，并把 buffer 地址填入 RX 描述符中，这样网卡在真正收到报文时就可以直接 DMA 到这些 buffer 里。
