IGB 网卡启动流程-igb_open（六）

igb_set_rx_mode

igb_configure() 中会调用 igb_set_rx_mode(netdev)，用于根据当前网卡的接收模式配置硬件接收过滤规则，比如混杂模式、多播过滤、单播地址表、VLAN 过滤等。具体实现如下所示：

igb_set_rx_mode() 主要负责配置网卡的接收过滤模式。函数首先根据 netdev->flags 判断网卡是否开启了混杂模式 IFF_PROMISC 或全多播模式 IFF_ALLMULTI，如果开启混杂模式，就设置 RCTL.UPE/MPE，允许接收所有单播和多播报文；如果只是全多播模式，则只打开多播混杂。否则驱动会把协议栈维护的多播地址列表写入硬件 MTA 表，把单播地址列表同步到 RAR 表，如果硬件表空间不足，就退化为对应的混杂接收模式。随后函数默认打开 VLAN 过滤 RCTL.VFE，但在混杂模式或 NETIF_F_RXALL 场景下会尝试放宽 VLAN 过滤，避免因为 VLAN 规则限制导致某些报文收不到。最后函数把这些模式写入 E1000_RCTL 寄存器，并配置最大接收包长 RLPML。

igb_setup_tctl

igb_setup_tctl() 主要是在初始化 igb 网卡的全局发送控制寄存器 TCTL。它先把默认可能开启的 Tx queue 0 关掉，避免旧状态影响后续队列配置；然后读取 E1000_TCTL，配置发送相关控制位，比如 PSP 让硬件自动给小于以太网最小帧长的报文补齐 padding。

igb_setup_mrqc

负责配置 igb 网卡的 RSS 接收分流能力，主要是写入 RSS hash key、配置 RETA 重定向表，并设置 MRQC 寄存器，让硬件可以根据报文 hash 把流量分散到不同的 RX 队列。具体代码如下所示：

igb_setup_mrqc() 主要是在配置 igb 网卡的 多接收队列/RSS 分流逻辑。它先生成一组 RSS hash key 并写入硬件 RSSRK 寄存器，然后根据当前可用的 RX 队列数初始化 RSS 重定向表 rss_indir_tbl，让 128 个 RETA entry 均匀映射到不同 RX 队列上。接着配置 RXCSUM，让硬件在接收描述符回写时带上 RSS hash 值，并根据 IPv4、IPv6、TCP、UDP 等报文字段选择 RSS hash 的输入字段。

igb_setup_rctl

igb_setup_rctl() 负责配置 igb 网卡的全局接收控制寄存器 RCTL。它会打开硬件接收功能，允许接收广播包，设置多播 hash 类型，开启 CRC 剥离和 jumbo frame 支持，同时默认不接收坏包。函数还会先关闭默认的 RX queue 0，避免队列未重新配置就被使用；如果开启 SR-IOV，则打开队列 drop，防止 VF 不提供描述符导致阻塞。

igb_configure_tx

igb_configure_tx() 负责初始化所有发送队列，把每个 TX ring 的描述符 DMA 地址、长度、head/tail 指针和队列控制参数写入硬件寄存器，为后续网卡发包做好准备。具体实现如下所示

igb_configure_tx() 先把每个 TX queue 的 TXDCTL 清零，确保队列处于关闭状态，然后等待硬件状态稳定，再逐个调用 igb_configure_tx_ring() 配置具体的发送 ring。每个 TX ring 会把描述符环的 DMA 基地址 TDBAL/TDBAH、长度 TDLEN、head/tail 指针 TDH/TDT 写到硬件寄存器里，同时保存 ring->tail，方便后续发包快路径直接更新 tail 通知硬件有新描述符。然后设置 TXDCTL 里的预取、主机读取、写回阈值，清空软件维护的 tx_buffer_info，最后置位 QUEUE_ENABLE 启用该发送队列。
