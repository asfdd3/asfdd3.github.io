igb 网卡初始化-igb_probe（二）

发表于 2026-06-14 更新于 2026-06-28 分类于 IGB网卡驱动 阅读次数： Valine：

igb_get_invariants_82575() 是 igb 驱动在 probe 早期调用的硬件识别函数。当 PCI 子系统发现一块和 igb 驱动匹配的网卡后，会调用 igb_probe()。在 igb_probe() 中，驱动先分配并初始化 net_device、igb_adapter 和 e1000_hw 等基础结构，然后调用 igb_get_invariants_82575() 读取 PCI device id 和相关硬件寄存器，判断当前网卡属于 82575、82576、82580、i350、i210 还是 i211 等型号，并设置 MAC、PHY、NVM、链路模式等固定属性。这里识别出来的信息会作为后续软件初始化、队列配置、中断初始化和链路初始化的基础。

igb_get_invariants_82575() 是 igb 驱动在 igb_probe() 早期调用的硬件识别函数。它首先根据 PCI device_id 判断当前网卡属于 82575、82576、82580、i350、i210、i211 还是 i354，并把结果保存到 hw->mac.type。这个类型后面会影响很多初始化逻辑，比如 MAC、NVM、PHY、SR-IOV 等不同硬件模块该走哪套处理方式。

接着函数读取 E1000_CTRL_EXT 寄存器，根据里面的 link mode 判断物理接口类型。默认先认为是普通铜口，如果发现是 1000BASE_KX，就认为是背板 SerDes；如果是 SGMII，还要判断后面是 MDIO 管理的外部 PHY，还是 I2C/SFP 模块；如果是 PCIE_SERDES 或 I2C 类型 SGMII，就会尝试读取 SFP EEPROM 确认介质类型。最后它会初始化 MAC、NVM、PHY 的参数和 ops；对于 82576、i350 这类支持 SR-IOV 的设备，还会初始化 mailbox。

这段 igb_init_mac_params_82575() 的作用是根据前面已经识别出来的 mac->type，初始化 MAC 层的一些固定参数和操作函数。它首先设置多播过滤表、单播 hash 过滤表，以及精确 MAC 地址表，然后函给 mac->ops 挂上对应的硬件操作函数，比如不同芯片使用不同的 reset 函数，最后，函数根据前面识别出来的 media_type 选择链路初始化函数。

igb_sw_init主要负责 软件资源初始化。它会先设置默认的 TX/RX ring 大小、中断节流 ITR、TX 清理上限、最大/最小帧长度，并初始化一些自旋锁。然后根据前面识别出来的硬件类型处理 SR-IOV VF 数量，默认标记支持 MSI-X，分配驱动维护的 MAC 地址表和 VLAN 过滤表。接着调用 igb_init_queue_configuration和 igb_init_interrupt_scheme，队列数量和中断/ring 资源的初始化。最后关闭中断。

igb_init_interrupt_scheme() 先调用 igb_set_interrupt_capability() 申请中断能力，优先尝试 MSI-X；在 MSI-X 模式下，驱动会根据 rss_queues 确定 RX/TX 队列数量，如果开启了 VF，TX 队列会限制成 1 个。如果 TX/RX 成对共用一个中断向量，就主要按 RX 队列数算；如果 TX/RX 分开处理，就 RX 队列数加 TX 队列数。除此之外，还会额外多申请 1 个 MSI-X 向量，用来处理 link status 这类非队列中断。pci_enable_msix_range() 成功后，内核会给 msix_entries[i].vector 填上真正的 Linux IRQ 号。随后为这些队列中断分配对应的 q_vector 管理结构，最后记录每个 ring 对应的硬件队列寄存器编号。
