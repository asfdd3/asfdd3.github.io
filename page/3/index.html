显示空闲链表

隐式空闲链表查找效率低，适合块数很少的情况；显式空闲链表把空闲块用前后指针连成链表，比如用双向链表，可以大大加快查找速度（只跟空闲块数量有关），如果用后进先出插入，释放块和合并可以是常数时间，但如果按地址排序虽然释放块需要找位置（慢一些），但内存利用率更高；缺点是显式链表需要额外的前后指针，块最小尺寸增大，增加了内部碎片。

与内存有关的错误

间接引用坏指针

  • 间接引用就是用指针去访问内存。
  • 坏指针就是：
    • 指向了不该访问的区域（比如没分配过的虚拟内存洞）
    • 或者指向了只读区域（但你却想写它）
    • 或者根本就是乱的值（未初始化、被覆盖、拼错……）

当你去解引用（访问 *p）一个坏指针时，操作系统会触发保护机制（段错误或保护错误），直接把程序干掉，防止你破坏别的内存。

虚拟内存简要总结

虚拟内存的核心作用

  • 每个进程都有独立的虚拟地址空间和页表，实现地址空间隔离和内存保护。
  • 操作系统通过页表把进程的虚拟地址映射到物理内存，实现内存管理的灵活性。

虚拟内存带来的好处

  • 简化链接：不同进程的代码和数据在虚拟空间中布局一致（比如代码总从同一个虚拟地址开始），链接器生成的可执行文件不依赖物理内存实际布局，简化了实现。
  • 简化加载：加载器只需分配虚拟页并做映射，实际数据只有在访问时才按需从磁盘加载到内存（即按需调页、懒加载）。
  • 简化共享：同一份物理内存（如操作系统内核、C标准库）可以被多个进程的虚拟空间同时映射，实现高效代码和数据共享。
  • 简化内存分配：进程申请新内存时，操作系统只需分配若干虚拟页，并将它们映射到任意空闲的物理页，不必保证物理上连续。

共享机制

  • 默认情况下，每个进程的虚拟空间互不影响，各自私有。
  • 需要共享的内容（如内核代码、共享库）可被多个进程的虚拟空间指向同一份物理内存。

内存映射 (memory mapping)

  • 操作系统允许把一组虚拟页映射到文件的任意位置，实现高效的文件访问与共享（如 mmap 系统调用）。

虚拟内存（一）

虚拟内存

概念：

为了更有效地管理内存并减少出错，现代系统提供了一种对主存的抽象，叫做 虚拟内存（VM）。 虚拟内存是硬件异常、地址翻译、主存、磁盘文件和内核软件的共同作用，为每个进程提供了一个大的一致且私有的地址空间。

虚拟内存提供三大核心能力：

  1. 高效利用主存
    将主存视为磁盘地址空间的高速缓存，仅保存活跃区域，按需在磁盘和主存之间调页，从而高效使用物理内存。

  2. 一致的地址空间
    给每个进程提供一致的虚拟地址空间，简化内存管理和程序设计。

  3. 地址空间隔离与保护
    防止进程间互相干扰，保障系统稳定性和安全性。

物理寻址与虚拟寻址

物理寻址（Physical Addressing）
主存由 M 个连续的字节单元组成，每个字节有唯一的物理地址（如 0, 1, 2, …）。
早期 PC、一些数字信号处理器、嵌入式微控制器和 Cray 超级计算机等直接使用物理地址，CPU 访问内存时直接生成物理地址，通过内存总线取出数据。

虚拟寻址（Virtual Addressing）
现代处理器一般使用虚拟寻址。CPU 生成的地址是虚拟地址（VA），需要通过**地址翻译（Address Translation）转换成对应的物理地址（PA）。
地址翻译由 CPU 的内存管理单元（MMU）**和操作系统配合完成。MMU 使用操作系统维护的查询表（如页表），动态把虚拟地址映射到物理地址，实现灵活的内存管理和保护。

地址空间

就是一个有序的、非负整数地址的集合，用来给数据单元（如字节）编号。

符号解析

符号解析就是链接器将“引用的符号”与“定义的符号”进行匹配的过程。

链接器如何解析多重定义的全局符号

链接器处理多个 可重定位目标文件（.o 文件）。

每个模块中可能会定义同名的全局符号（如变量、函数）。

符号分为两类：

  • 强符号（strong）：函数、已初始化的全局变量（如 int x = 1;）
  • 弱符号（weak）：未初始化的全局变量（如 int x;）
规则编号 描述
规则 1 多个强符号同名 → 报错，链接失败（冲突无法解决）。
规则 2 强符号 vs 弱符号同名 → 选择强符号，忽略弱的。
规则 3 多个弱符号同名 → 任意选择一个（通常选择第一个出现的）。

静态库

静态库（static library）：是由多个可重定位目标文件（.o）打包成的一个 .a 文件，供链接器使用。

链接

链接（Linking）是一个将多个代码和数据片段（来自编译器生成的目标文件）收集并组合为一个单一文件的过程，这个文件可以被加载到内存中执行。

链接的目标文件可以是：

  • 可执行文件
  • 共享库（动态库）
  • 或中间的可重定位文件

链接可以发生在不同的阶段：

链接阶段 说明
链接时（Link-time） 最常见的，指用 ld 或 gcc 把多个 .o、.a、.so 链接为一个可执行文件或库。
加载时（Load-time） 程序被操作系统加载到内存时，动态链接器（如 /lib64/ld-linux.so.2）把 .so 文件装载并绑定符号。
运行时（Run-time） 程序运行过程中自行调用 dlopen() / dlsym() 等函数动态加载库并解析符号。

静态链接

静态链接是指链接器将所有需要的代码、数据和库函数一次性合并到一个最终的可执行文件中，使这个文件在运行时不再依赖外部库

struct sock

发表于 2025-06-22 分类于 网络协议栈源码学习 Valine：

sock 结构体在 Linux 网络协议栈中的作用非常核心，它本质上是用于管理传输层连接状态和控制信息的内核数据结构

sock 结构体的主要功能：

1. 管理传输层连接状态

  • 记录 socket 状态，如 TCP 的 ESTABLISHED、LISTEN、SYN_SENT 等；
  • 维护连接生命周期。

2. 数据收发管理

  • 接收队列：sk_receive_queue
  • 发送队列：sk_write_queue
  • Backlog 队列：sk_backlog

3. 缓冲区和流控

  • 接收缓冲区大小（sk_rcvbuf）
  • 发送缓冲区大小（sk_sndbuf）
  • 控制数据收发速率，避免内存耗尽。

4. 定时器与超时管理

  • sk_timer：重传、超时的定时器；
  • 如 TCP 的重传定时器（RTO）、保活定时器（keepalive）。
