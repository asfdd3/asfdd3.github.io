struct sock

发表于 2025-06-22 分类于 网络协议栈源码学习 阅读次数： Valine：

sock 结构体在 Linux 网络协议栈中的作用非常核心，它本质上是用于管理传输层连接状态和控制信息的内核数据结构

sock 结构体的主要功能：

1. 管理传输层连接状态

  • 记录 socket 状态，如 TCP 的 ESTABLISHED、LISTEN、SYN_SENT 等；
  • 维护连接生命周期。

2. 数据收发管理

  • 接收队列：sk_receive_queue
  • 发送队列：sk_write_queue
  • Backlog 队列：sk_backlog

3. 缓冲区和流控

  • 接收缓冲区大小（sk_rcvbuf）
  • 发送缓冲区大小（sk_sndbuf）
  • 控制数据收发速率，避免内存耗尽。

4. 定时器与超时管理

  • sk_timer：重传、超时的定时器；
  • 如 TCP 的重传定时器（RTO）、保活定时器（keepalive）。

5. 协议特定操作**

  • 每个协议（TCP/UDP）定义自己的操作集，绑定到 sock 的 sk_prot 字段；
  • 例如 TCP 操作集合为 tcp_prot，UDP 为 udp_prot；
  • 提供 sendmsg()、recvmsg()、close() 等接口。

6. 连接路由与缓存

  • 加速报文发送时的路由查询。

7. 等待队列管理

  • sk_wq：实现 select()、poll()、epoll() 等机制；
  • 用户空间程序等待 socket 可读/可写事件时使用。
