socket I/O

输出系统调用

send、sendto 和 sendmsg 都是用于发送数据的系统调用，面向连接的协议通常使用send(例如tcp或者已经connect的udp)，而无连接的协议通常使用send_to而sendmsg支持发送时设置多个缓存区，也就是说一次调用可以发送多个数据包。

参数名 类型 描述 注意事项
sockfd int 已连接的 socket 文件描述符 必须是通过 connect() 建立连接的 TCP socket 或已连接的 UDP socket
buf const void * 待发送数据的缓冲区地址 用户态指针，指向要发送的数据（如字符串、二进制数据等）
len size_t 要发送的数据长度（字节数） 实际发送的数据量可能小于此值（需检查返回值）
flags int 控制发送行为的标志位 多个标志可通过按位或 | 组合（如 MSG_DONTWAIT | MSG_NOSIGNAL）

接收系统调用

recv,recvfrom,recvmsg三个接收系统调用与发送系统调用类似，不同的地方就是数据流向是相反的。

系统调用 适用场景 关键功能 典型用途
recv 已连接的套接字（如 TCP） - 从已建立连接的套接字接收数据
- 不支持获取发送方地址		 TCP 数据接收
recvfrom 无连接套接字（如 UDP） - 接收数据包
- 可获取发送方地址（struct sockaddr）		 UDP 数据接收
recvmsg 所有套接字（最通用） - 支持多缓冲区（struct iovec）
- 支持控制信息（cmsg）
- 可获取发送方地址		 高级场景：
- 文件描述符传递
- 接收 TTL/接口信息
